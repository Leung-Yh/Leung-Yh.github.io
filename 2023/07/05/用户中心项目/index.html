@Resource 与 @Autowired 的区别

@Resource 会默认按照 Java 的名称去注入属性，如果是 @Autowired 的话，只会按照类型去注入属性，所以一般用 @Resource 来自动注入

简单说明 Cookie 和 Session

Cookie 是一种缓存机制，Session 是会话机制

  • 以最常见的登录案例讲解 cookie 的使用过程：

    • 首先用户在客户端浏览器向服务器首次发起登录请求
    • 登录成功后，服务端会把登录的用户信息设置在 cookie 中，并将 cookie 返回给客户端浏览器
    • 客户端浏览器接收到 cookie 请求后，会把 cookie 保存到本地（可能是内存，也可能是磁盘）
    • 以后再次访问该 web 应用时，客户端浏览器就会把本地的 cookie 带上，这样服务端就能根据 cookie 获得用户信息了

  • 同样以登录案例为例子讲解 session 的使用过程

    • 首先用户在客户端浏览器发起登录请求
    • 登陆成功后，服务端会把用户信息保存在服务端，并返回一个唯一的 session 表示给客户端浏览器
    • 客户端浏览器会把这个唯一的 session 标识保存起来
    • 以后再次访问 web 应用时，客户端浏览器会把这个唯一的 session 标识带上，这样服务端就能根据这个唯一标识找到用户信息。

  • 疑问1：把唯一的 session 标识返回给客户端浏览器，然后保存起来，以后访问时带上，这难道不是cookie吗

    解答：没错，session 只是一种会话机制，在许 web 应用中，session 机制就是通过 cookie 来实现的。也就是说它只是使用了 cookie 的功能，并不是使用 cookie 完成会话保存。与 cookie 在客户端保存会话的机制相反，session 通过 cookie 的功能把会话信息保存到了服务端

  • 疑问2：session 和 cookie 有什么区别

    • （1）cookie 是浏览器提供的一种缓存机制，它可以用户维持客户端与服务端直接的会话
    • （2）session 指的是维持客户端与服务端会话的一种机制，它可以通过 cookie 实现，也可以通过别的有段实现
    • （3）如果用 cookie 实现会话，那么会话会保存在客户端浏览器中
    • （4）而 session 机制提供的会话是保存在服务端的

Nodejs新版本引起的问题

问题描述：

解决方法：在需要执行的命令之前加入 SET NODE_OPTIONS=--openssl-legacy-provider

参照：nodejs新版本引起的：digital envelope routines::unsupported

运行报错：Error: error:0308010C:digital envelope routines::unsupported

  • 报错原因：NodeJs 版本比较高，不支持该命令

  • 解决方法：

    • 简单粗暴法：在命令前加上 SET NODE_OPTIONS=--openssl-legacy-provider &&

      即 SET NODE_OPTIONS=--openssl-legacy-provider && 原命令
